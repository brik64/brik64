# BRIK64 Alpha Launch Coordination

This document translates the current product contract into website work for
`brik64.com`.

## Active contract

- public CLI name: `brik64`
- internal compiler/testing name: `brikc`
- public ecosystem state: `Alpha` on `L4+`
- platform state: `Beta` on `L4+`
- `Public RC` gate: `L4+` PCD self-lift and multi-target replay
- `1.0` gate: platform migrated to `L5+` and `L5+` PCD self-lift closed

## What the site can say now

- BRIK64 has a public alpha ecosystem.
- The platform is available as a beta surface on `L4+`.
- `L5+` is a future premium remote engine.
- `brik64.ai` is the future AI gateway domain.
- `brik64.org` and `brik64.net` are defensive redirect domains in this phase.

## What the site must not say

- Do not call the total product `1.0`.
- Do not imply that `L4+` self-lift alone closes `1.0`.
- Do not present `L5+` as a local downloadable engine.
- Do not promise VS Code or full LSP availability as a launch-day requirement.
- Do not use `brikc` as the public product name.

## Page groups to align first

1. home
2. cli/download
3. docs handoff pages
4. platform
5. ai / gateway pages
6. changelog and status surfaces

## Coordination dependencies

- Infra must define the active domain and redirect authority.
- Docs must remain canonical on `docs.brik64.com`.
- Platform copy must reflect `L4+` current and `L5+` remote future.
- SDK copy must not overclaim certification or `Public RC`.

## Release checklist for the website lane

- naming aligned to `brik64`
- `Alpha` / `Public RC` / `1.0` ladder matches docs
- no `docs.brik64.dev` links on active pages except explicit historical context
- `brik64.ai`, `.org`, and `.net` handled intentionally in copy
- changelog and status wording match the docs contract
