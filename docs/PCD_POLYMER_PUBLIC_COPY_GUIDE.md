# PCD And Polymer Public Copy Guide

Date: 2026-05-01

## Public Definition

For public website copy:

- `PCD`: a blueprint for one bounded logic circuit.
- `Polymer`: a reusable composition of multiple PCDs.

Do not use these terms interchangeably.

## Recommended Explanation

```text
BRIK64 turns critical logic into PCD blueprints. Individual PCDs can be
compiled and certified. Teams can then compose multiple certified PCDs into
polymers: reusable, versioned logic packages with dependency graphs, hashes and
evidence.
```

## Claim-Safe Framing

Safe:

- "PCDs make critical logic reviewable."
- "Polymers package multiple PCDs into reusable workflows."
- "Certificates and hashes describe the verified boundary."

Avoid:

- "A PCD is a polymer."
- "A polymer proves the whole application correct."
- "Certified app" unless the application-level boundary and evidence are shown.

## Website IA Implication

Future product pages should distinguish:

- `/pcd`: individual blueprints and compilation.
- `/polymers`: reusable compositions and registry packaging.
- `/registry`: discovery, versioning, hashes and evidence.
