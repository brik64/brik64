# Hero Image Prompt System (BRIK64)

Use this base prompt to keep a consistent visual language across `home`, `industries`, and `product`.

## Base prompt (master template)

`BRIK64 {surface_type} hero background, ultra-wide 1792x1024, dark infra-grade aesthetic, left side formal blueprint/circuit drafting language with cyan linework and clean text-safe negative space, smooth cinematic blend into right side AI-generated realistic real-world scene that reflects {domain_context}, restrained amber accents, subtle vignette, no logos, no brand marks, no readable text, no watermark, high detail, low noise in headline zone.`

## Composition rules

- Home + industries:
  - Blueprint (left/center): `60–70%`
  - Real-world scene (right): `30–40%`
  - Keep center-left readable for headline + CTA.
- Product pages:
  - Blueprint (left/center): `70–75%`
  - Real-world scene (right): `25–30%`
  - Keep center cleaner than industries to protect longer product hero copy.

## Domain context tokens

- `science`: reproducibility lab, calibration rigs, instrumentation, observatory compute
- `government`: oversight operations, deterministic routing, case-state review
- `healthcare`: dose/escalation review, clinical workflow stations, device-safe context
- `platform`: governed workspace + registry operations
- `cli`: terminal workflow + engineering workstation
- `pcd`: formal blueprint design environment
- `transpiler`: multi-language translation pipeline operations
- `lifter`: legacy-to-canonical migration environment
- `sdks`: integration engineering workspace across runtimes
- `registry`: package discovery and publication operations
- `compliance`: audit trail + evidence review context
- `bpu`: secure hardware module/chip integration lab
- `features`: product capability matrix as operational system
- `foundations`: information-theory and formal-methods research environment
- `home-hero`: formal-to-real-world bridge (software-to-physical impact)
- `home-digital-circuitality`: abstract theoretical backbone with production atmosphere
